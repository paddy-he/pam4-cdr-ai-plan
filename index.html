PAM4 CDR AI 辅助设计执行计划

目标是在一个月内完成 CDR 架构选取与系统 model 验证，同时用一个简单 DCC block 打通 ADT / virtuoso-bridge-lite / 仿真自动化 / 参数优化流程。

1. 项目目标

主线 A：PAM4 CDR 架构与模型

一个月内完成从文献调研到架构选择，再到 Python/MATLAB 行为级模型验证的闭环。

ISSCC 调研 架构矩阵 Jitter Budget System Model

并行线 B：DCC Block AI 设计试跑

选一个常见 Duty-Cycle Corrector 结构，重点验证 AI 工具链如何加速 tb 搭建、参数搜索、仿真结果提取和设计记录。

ADT virtuoso-bridge-lite Spectre/Ocean Optimization
执行原则：主线 A 决定真正的 CDR 方向；并行线 B 不追求 DCC 最优性能，追求把 AI 辅助模拟设计流程跑通，形成可复用方法。

2. 一个月时间表

第 1 周

主线 A：建立文献库，完成 ISSCC/JSSC/VLSI 关键词检索表，初步筛出 20-40 篇相关论文。

并行线 B：确认本地 Cadence、PDK、Spectre、ADT、virtuoso-bridge-lite 是否可用，定义 DCC testbench。

第 2 周

主线 A：完成 56G/112G/224G 架构趋势对比，提取关键指标和架构 trade-off。

并行线 B：完成 DCC schematic 初版、tb 初版、基础 transient 仿真和 duty-cycle 测量脚本。

第 3 周

主线 A：建立 PAM4 链路行为模型，包括 channel、CTLE/FFE/DFE、sampler、jitter 和 CDR loop。

并行线 B：接入参数 sweep / 优化器，输出 pass/fail summary 和下一轮 sizing 建议。

第 4 周

主线 A：完成候选架构 model 验证，形成架构选择报告和下一阶段电路 block 分解。

并行线 B：总结 AI 设计流程，沉淀 DCC 自动化模板，迁移到 CDR 子模块设计。

7. 当前风险与待确认

  • PDK / Virtuoso / Spectre 是否可用：决定并行线 B 能否直接接入真实晶体管仿真。
  • 目标速率尚未定死：主线 A 暂时按 56G/112G/224G 三档比较，但第 2 周末需要收敛优先级。
  • ADT / virtuoso-bridge-lite 能力边界未知：先用 DCC block 验证工具连接、netlist 生成、tb 操控、结果解析。
  • 224G 风险较高：若最终目标是 224G，架构和模型阶段必须更重视 ADC/DSP、time-interleaving、clock distribution 和 calibration。
